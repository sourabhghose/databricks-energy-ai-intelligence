## Overview

Databricks Genie AI/BI provides a natural language interface directly to Delta tables. Users type a question in plain English; Genie translates it to SQL, executes it against the Gold tables, and returns results as a table or chart â€” with the generated SQL visible for transparency.

Energy Copilot configures **16 Genie spaces**, each optimised for a specific domain with curated semantic metadata (table and column descriptions, glossary terms, and benchmark questions).

## Adding New Genie Spaces

To add a new Genie space:

1. Identify the Gold tables to include
2. Ensure all table and column COMMENTs are comprehensive
3. Prepare 15 sample benchmark questions
4. Prepare a domain glossary (10+ terms)
5. Add the space configuration to `setup/04_create_genie_spaces.py`
6. Run the setup script:

```bash
databricks jobs run-now --job-id <SETUP_JOB_ID> \
  --python-params '["create_genie_only"]' \
  --profile=my-profile
```

## Genie vs Copilot

| Feature | Genie AI/BI | AI Copilot |
|---------|------------|-----------|
| Interface | Direct question â†’ SQL â†’ result | Conversational chat |
| SQL visibility | Yes (show SQL) | No (abstracted) |
| Custom tool calling | No | Yes (58 tools) |
| Multi-step reasoning | Limited | Yes (agentic loop) |
| ML model access | No | Yes |
| Real-time data | Yes (direct table access) | Yes (via tool calls) |
| Best for | Direct data queries, ad hoc SQL | Complex analysis, trading actions, narratives |

<Aside type="note">
  Genie space IDs are resolved dynamically at app startup. A background thread queries the workspace Genie API and matches spaces by title, updating the hardcoded IDs in-place. This means Genie spaces work correctly after a fresh deploy even if space IDs change. If the API is unreachable, the app falls back to the hardcoded IDs.
</Aside>

  Genie space permissions require specifying the `applicationId` UUID of the App service principal (not the numeric SP ID) â€” see the post-deploy setup guide for details.
