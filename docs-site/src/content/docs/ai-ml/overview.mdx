## The Databricks AI Story

Energy Copilot demonstrates the full Databricks AI stack working together:

1. **Foundation Model API (FMAPI)**: Claude Sonnet 4.5 routed through Databricks, keeping API calls within the network perimeter
2. **MLflow Model Registry**: all 5 ML models trained, tracked, versioned, and deployed via MLflow
3. **Model Serving**: inference endpoints for real-time model predictions
4. **Genie AI/BI**: natural language SQL over Gold Delta tables via 12 Genie spaces
5. **Unity Catalog SQL Functions**: 14 UC functions provide the AI agent with structured access to data
6. **Vector Search** (RAG): AEMO market rules indexed for semantic retrieval by the copilot

## MLflow Experiment Tracking

All ML models are tracked in MLflow experiments within Unity Catalog:

| Experiment | Model Type | Target Variable | Regions |
|-----------|-----------|----------------|---------|
| `price_forecast` | LightGBM | NEM spot price | All 5 NEM |
| `demand_forecast` | LightGBM | Operational demand | All 5 NEM |
| `wind_forecast` | LightGBM | Wind generation | All 5 NEM |
| `solar_forecast` | LightGBM | Solar generation | All 5 NEM |
| `anomaly_detection` | IsolationForest | Price anomaly | NEM-wide |

Each experiment tracks:
- Training hyperparameters (Optuna 50-trial HPO)
- Evaluation metrics (MAE, MAPE, AUC, F1)
- Feature importances
- Chronological train/val/test split boundaries (stored as tags)
- Model artifacts and registered model alias

## Model Serving

The following Model Serving endpoints are configured:

| Endpoint Name | Model | Purpose |

Model Serving endpoints use serverless compute and auto-scale to zero when not in use.

## How AI/ML Capabilities Work Together

```
User Question (Copilot)
        │
        ▼
Claude Sonnet 4.5 (FMAPI)
        │
        ├──→ get_price_forecast    ──→ Model Serving (LightGBM)
        │                                     │
        │                                     ▼
        ├──→ get_generation_mix    ──→ Gold Table (Lakebase)
        │
        ├──→ get_stpis_metrics     ──→ Gold Table (SQL Warehouse)
        │
        ├──→ get_asset_health      ──→ Gold Table + Model Serving
        │
        └──→ search_market_rules   ──→ Vector Search Index
```

The copilot's agentic loop orchestrates calls to multiple tools in sequence or parallel, synthesising the results into a coherent, contextualised response.

## Genie vs Copilot

Both the Copilot and Genie spaces provide natural language access to data, but serve different purposes:

| Feature | AI Copilot | Genie AI/BI |
|---------|-----------|------------|
| Interface | Conversational chat | Direct question → SQL → result |
| Access to real-time data | Yes (via tool calls) | Yes (direct table access) |
| Custom tool calling | Yes (51 tools) | No |
| Multi-step reasoning | Yes (agentic loop) | Limited |
| Best for | Complex analysis, narratives | Direct data queries |
| SQL visibility | No (abstracted) | Yes (show SQL) |
